# Testing Strategy

Status: TEMPLATE — Architect must complete during Sprint 0.

## Required levels

- schema validation;
- semantic model behavior;
- normalization and provenance;
- rendering DOM assertions;
- stable snapshots where appropriate;
- corpus regression;
- transposition;
- accessibility checks;
- invalid and contradiction cases;
- user learning protocol outside automated software tests.

## Testing rule

Every semantic construct requires:

1. valid schema example;
2. invalid schema or semantic example;
3. model behavior test;
4. normalization test when applicable;
5. rendering test when visible;
6. corpus regression case when discovered through real music.

Visual snapshots may supplement but may not replace semantic assertions.
