# Product Roadmap

## Milestone 0 — Requirements baseline

Objective: establish authoritative product documents, stable IDs, governance, and agent roles.

Exit criteria:

- Constitution approved;
- requirements indexed;
- decisions recorded;
- assumptions separated;
- architecture prompt ready.

Status: Complete.

## Milestone 1 — Architecture Sprint 0

Objective: produce an implementation-ready technical design without production code.

Deliverables:

- architecture overview;
- canonical model specification;
- repository and package structure;
- rendering pipeline;
- pattern engine design;
- testing strategy;
- ADRs;
- Sprint 1 engineering plan;
- traceability update.

## Milestone 2 — Canonical model skeleton

Objective: implement versioned schema, domain types, validation, pitch strategy boundary, and minimal fixtures.

## Milestone 3 — Minimal renderer

Objective: render full, harmonic, and role-isolated views from canonical or normalized data without mutation.

## Milestone 4 — Golden corpus

Objective: encode representative lawful fixtures and establish regression, vocabulary, and provenance reports.

## Milestone 5 — Competing experiments

Objective: compare pitch representation, beat presentation, repetition, learning chunks, pattern vocabulary, and familiar-shape hints.

## Milestone 6 — Learning validation

Objective: compare the prototype with chord sheets and traditional notation through repeatable learning trials.

## Milestone 7 — Language consolidation

Objective: choose or design human-authoring syntax only after semantic and learning evidence exists.
